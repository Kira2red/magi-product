# 产品大师 — 部署说明

## 1. 文件清单

```
product-master/
├── SKILL.md              # Controller 调度逻辑（必须）
├── lead-pm-prompt.md     # 主力产品经理子 Agent prompt（必须）
└── reviewer-prompt.md    # 纠错大师子 Agent prompt（必须）
```

## 2. 部署步骤

### 步骤 1：复制文件

将整个 `product-master/` 文件夹放到以下任一位置：

- `~/.openclaw/workspace/skills/product-master/` （推荐）
- `~/.openclaw/skills/product-master/`

```bash
cp -r product-master ~/.openclaw/workspace/skills/
```

### 步骤 2：添加路由规则

⚠️ **这是最关键的一步，不配不会触发。** 编辑 `~/.openclaw/prompts/workspace-guide.md`。如果该文件不存在，创建它：

```markdown
---
description: 工作区文件结构与命名规范
alwaysAttach: true
---

## 产品大师路由规则

当用户输入满足以下任一条件时，必须加载产品大师 Skill（路径：~/.openclaw/workspace/skills/product-master/SKILL.md）：

1. **用户明确说"启用产品大师"** → 强制加载
2. **用户要求产出产品方案、PRD、功能清单、交互方案、场景设计，且期望有质量把控** → 优先加载产品大师

加载后你成为指挥者（Controller），按 SKILL.md 的流程执行：
- 第 1 步：派生子 Agent 做需求检查 + 复杂度评估 + 子任务拆分
- 第 2 步：逐子任务执行评审循环（Lead PM 产出 → Reviewer 审查 → PM 修订 → 再审查，直到收敛）
- 第 3 步：整合输出最终方案

**你永远不自己写方案**——用 sessions_spawn 或 Task 工具派生子 Agent。
```

`alwaysAttach: true` 确保每条消息都加载这些规则，否则只有会话开始加载一次。

### 步骤 3：重启 OpenClaw

新 skill 需要重启才能被 OpenClaw 扫描到。

## 3. 使用方式

在新会话中直接用自然语言触发（**不要用 `/`，OpenClaw 不支持自定义 slash command**）：

```
启用产品大师，帮我设计一个车载语音助手的交互方案
```

或者直接描述产品规划任务（路由规则会自动匹配）：

```
我需要给 Robotaxi 设计一套座舱交互方案，前排大屏 + 后排触控小屏
```

## 4. 可选增强

- 如果有 `2red-product-monster-prd` skill，Lead PM 产出的方案会更有章法。没有也不影响运行
- 仪表盘：`server.py` + `dashboard.html` 提供实时状态可视化（见仪表盘目录）

## 5. 模型要求

建议使用 DeepSeek V4 Pro 或同等级模型。三个 Agent 统一用同一模型。

## 6. 常见问题

**Q: 说"启用产品大师"但没触发？**
A: 检查步骤 2 路由规则是否配置且 `alwaysAttach: true`，重启 OpenClaw。

**Q: 走到「待确认事项」后一直不动？**
A: 正常。Controller 在等你回复补充信息，直接在对话里回答即可。

**Q: 循环看起来不会停了？**
A: 回复「收敛」强制结束当前子任务，或「到哪了」查看进度。
